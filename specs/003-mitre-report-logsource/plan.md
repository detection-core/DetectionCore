# Implementation Plan: MITRE Heatmap, Detection Report, and LogSource Coverage Fix

**Branch**: `003-mitre-report-logsource` | **Date**: 2026-03-22 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/003-mitre-report-logsource/spec.md`

## Summary

DetectionCore needs three improvements: (1) hierarchical log source matching to fix false "not covered" status on rules, (2) a MITRE ATT&CK matrix heatmap page showing detection gaps, and (3) a comprehensive detection status report page. The approach uses in-memory lookup dicts for log source matching, a static MITRE ATT&CK reference dataset, custom CSS grid for the heatmap, and browser-native print for reports.

## Technical Context

**Language/Version**: Python 3.11 (backend), TypeScript 5.x (frontend)
**Primary Dependencies**: FastAPI, Beanie ODM, pySigma (backend); React 18, Recharts, TanStack Query, Tailwind CSS (frontend)
**Storage**: MongoDB via Beanie ODM
**Testing**: Manual verification via browser (see quickstart.md)
**Target Platform**: Windows 10 (dev), Docker (prod)
**Project Type**: Web application (FastAPI + React SPA)
**Performance Goals**: Report generation < 5 seconds, heatmap render < 2 seconds
**Constraints**: No external MITRE API calls at runtime; static reference data only
**Scale/Scope**: ~500 rules, ~200 MITRE parent techniques, ~25 unique log source combinations

## Implementation Phases

**Files**: `rule.py`, `log_sources.py`, `scoring_engine.py`, `endpoints.ts`, `LogSources.tsx`, `Rules.tsx`

1. Add `log_source_match_type: Optional[str] = None` to `DetectionRule` model
2. Rewrite `_refresh_rule_log_availability()` with three-tier lookup:
   - Build `exact_keys`, `cat_product_keys`, `product_keys` dicts from LogSource table
   - For each rule: try exact в†’ category+product в†’ product-only
   - Set `log_source_available` and `log_source_match_type`
3. Simplify `_score_log_availability()` to read cached match type from rule
4. Add `POST /api/log-sources/auto-discover` endpoint
5. Add `GET /api/log-sources/coverage-summary` endpoint
6. Add `RuleSummaryOut.log_source_match_type` to rules router response
7. Frontend: Add auto-discover button and coverage banner to LogSources.tsx
